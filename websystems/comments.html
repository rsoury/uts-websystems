Structure

The website is structured primarily with tables and table cells, similarly to the way Marketing Emails are structured today because of the requirement to use HTML4 and CSS2 which do not have any sort of responsive grid based functionality like CSS3's Flexbox or @media selectors. Within each of these cells is our content, usually wrapped in a parent container. 

                      <table class="grey-border-bottom" width="100%" cellpadding="0" cellspacing="0">

                         <tr>

                            <td class="border-bottom-cell">

                               <!-- First Cell in First Row -->

                            </td>

                            <td class="border-bottom-cell">

                               <!-- Second Cell in First Row -->

                            td>

                         </tr>

                        

                         <tr>

                            <td class="border-right-cell half-cell">

                               <!-- First Cell in Second Row -->

                            </td>

                            <td class="half-cell">

                               <!-- Second Cell in Second Row -->

                            </td>

                         </tr>

                      </table>

I used many classes for even the smallest CSS styles, however there are primary classes that were used many times in one webpage which is essentially what classes are for. Classes are reusable labels representing a set of styles. When applied to an element, the element inherit the styles defined in the CSS class.

Class names are meant to be self explanatory. In the example below, it is clear that .main-nav-anchor is a class abbreviation for "Main Navigation Anchor" which is correct described as it defines the styling for the main navigation anchor elements on the web page. As already described, the reason why they're classes ".main-nav-anchor" and not ids "#main-nav-anchor" is because classes are more for style reusability and because multiple elements with the same id contradicts the purpose of an identification attribute. 

                      
                        .main-nav-anchor{

                            text-decoration: none;

                            color: #8c6eff;

                            border: 1px solid #8c6eff;

                            display: block;

                            font-size: 18px;

                            height: 40px;

                            line-height: 2;

                            margin: 0 10px;

                        }

                        .border-bottom-cell{

                            border-bottom: 5px solid rgb(240, 240, 240);

                        }

                        .border-right-cell{

                            border-right: 1px solid rgb(220, 220, 220);

Aesthetics

My aim was to strike a very ergonomic, simple and easy-to-understand layout. Each of the subpages carries a very similar layout ensuring that there is consitency between all the subpages. The header is depicted as it usually is across a multitude of websites in order to keep a web friendly layout. Title on the left top of the page and the main navigation on the top right.

On the contrary, the index page has a layout where everything is centered in order to direct the attention of the reader from top to bottom. Complementing this concept, the title of the website and main image are aimed to be the first thing new visitors see, and therefore appear above the navigation items. Centralised navigation is not something new or unergonomic as it depict more a of a splash page, where an understanding of the general idea behind the website is as important as the navigation around the web page.

Accessibility

Colour was selected to appeal to colour blind individuals. Complementary colours purple and light yellow establishes a significant distinguishment.

Correct placement of each element ensures that there are not any elements that are hiding behind the other, and that all elements which serve a purpose to the user, are accessable visually and interactively. If a user is visually impaired and is unable to clearly understand the functionality of an element, then that demonstrates poor accessability and that is, I believe, what I have prevented.
